## What Schema Markup Actually Is: The Code Behind Rich Results

Schema markup is code you add to your page that tells search engines exactly what each piece of information means. A product name is not just text. With schema, it becomes a typed entity Google can index, compare, and display in enhanced search features. It is code added to your website that transforms unstructured content into machine-readable data.

The code takes one of three forms. JSON-LD is the preferred form, as Google recommends it. You embed it in a script block in your page head or body. Microdata uses HTML tags (called microdata attributes) inline with your content. RDFa works similarly but uses a different attribute set. JSON-LD wins because it separates the data markup from the presentation layer, which makes implementation cleaner and maintenance simpler. All three forms reference the schema.org vocabulary, which defines schema types for everything from articles to software applications to FAQ pages. When you implement schema markup, you are writing in this language so machines can parse your page the way a human reader would. Every schema type has a set of properties that describe the entity, and those properties are the structured signals search engines use to generate rich results that can drive qualified traffic to your site.
